>_ 02 // DATA SOURCES

KERNEL DRIVERS & ENDPOINTS

// ENDPOINT 01
PROCESS ENDPOINT
Microsoft-Windows-Kernel-Process
Collects data via ETW (Event Tracing for Windows) session. Monitors all process creation, termination, and thread activity at the kernel level.
GUID: 22FB2CD6-0E7B-422B-A0C7-2FAD1FD0E716
// ENDPOINT 02
NETWORK ENDPOINT
Npcap (NDIS filter driver) + IP Helper API
Captures raw packets via pcap_loop on all network adapters. No pure ETW — uses deep packet inspection at the NDIS filter driver level.
Npcap NDIS Packet Filter Driver
// ENDPOINT 03
APPLICATION LOG
PowerShell ETW + WMI-Activity + Security & Defender
Collects via ETW + EvtSubscribe (Windows Event Log). Monitors PowerShell execution, WMI activity, and Security/Windows Defender event channels.
Microsoft-Windows-PowerShell // Microsoft-Windows-WMI-Activity
// ENDPOINT 04
FILE ENDPOINT (FIM)
Microsoft-Windows-Kernel-File
File Integrity Monitoring via ETW session. Tracks file create, write, delete, rename events at the kernel filter driver level.
GUID: EDD08927-9CC4-4E65-B970-C2560FB5C289
// ENDPOINT 05
USB ENDPOINT
WM_DEVICECHANGE (PnP) + SetupDi APIs
Uses Win32 hidden window to capture PnP kernel notifications + ReadDirectoryChangesW. Detects USB device insertion, removal, and storage access.
PnP Kernel Notifications + SetupDi API
Endpoint Kernel / Data Source Collection Method
1. Process Endpoint Microsoft-Windows-Kernel-Process provider ETW session (GUID: 22FB2CD6-0E7B-422B-A0C7-2FAD1FD0E716)
2. Network Endpoint Npcap (NDIS packet filter driver) + IP Helper API pcap_loop on network adapters (no pure ETW)
3. Application Log Microsoft-Windows-PowerShell + Microsoft-Windows-WMI-Activity (ETW) + Security & Windows Defender channels ETW + EvtSubscribe (Windows Event Log)
4. File Endpoint (FIM) Microsoft-Windows-Kernel-File provider ETW session (GUID: EDD08927-9CC4-4E65-B970-C2560FB5C289)
5. USB Endpoint WM_DEVICECHANGE (PnP kernel notifications) + SetupDi APIs Win32 hidden window + ReadDirectoryChangesW
>_ 03 // PROCESSING PIPELINE

ARCHITECTURE LAYERS

LAYER 2
PROCESSING DASHBOARD
🌐

GLOBAL FILTER

Applies system-wide filtering rules before events enter the correlation engine. Reduces noise and focuses on security-relevant events.

🔗

CORRELATOR

Takes multiple logs from all endpoints and correlates them using common key points — grouping related events into unified security incidents.

🤖

AI / OPENCTI

Validates correlated events using threat intelligence. Performs pattern matching against known attack signatures and threat actor TTPs.

LAYER 1
INTERFACE DISPLAY
📋

ALL .EXE PROCESSES DISPLAYED

All running processes and their associated .exe files are displayed in the interface with full process tree visibility and metadata.

📊

REAL-TIME MONITORING

Live telemetry streams from all 5 endpoints, presented in a unified dashboard view with timeline and severity-based sorting.

🚨

ALERT SURFACE

Correlated and AI-scored alerts surfaced to SOC analysts with full context, predicted next steps, and vulnerability mapping.

>_ 04 // INTELLIGENCE ENGINE

SOC DASHBOARD & AI ANALYSIS

🧠
AI MODEL — SCORE ALLOCATION

By training the AI model on historical attack data, the system allocates a threat score to every log entry and data event. It identifies pattern-based attacks in real-time, ranking events by severity and confidence level.

🔮
OPENCTI — THREAT PREDICTION

Using OpenCTI threat intelligence data, the system goes beyond detection to predict potential threats before they fully materialize. Historical TTP correlation enables proactive defence postures.

🛡️
WITH OPENCTI, THE SYSTEM CAN:
  • Detect Suspicious Activity: Analyse threat feeds, logs, and telemetry data to identify potential security issues before they escalate into full breaches.
  • Predict Threats: Use machine learning, pattern recognition, and analytics to forecast potential attacks or vulnerabilities specific to your environment and endpoint context.
⚡
AI FINAL ANALYSIS — WHAT THE ENGINE SURFACES
  • Based on all telemetry sources, the AI flags suspicious activities and events with confidence scores, full context, and source endpoint attribution.
  • Identifies vulnerable areas and loopholes in the system — predicting which attack vectors are most likely to be exploited given the current environment state.
  • If a virus or malware is recognised, the AI attempts to predict its next step — enabling SOC analysts to get ahead of the threat and contain it before propagation.
