01 / Prerequisite: API Keys

To fully utilize the Layer 3 Threat Intelligence Analyzer, you must link the system to real-world threat databases.

Before running the backend, obtain your personal API keys from AbuseIPDB and VirusTotal and insert them into the environment variables / backend configuration.

02 / Layer 1: Master Control Node (C++)

This folder contains the core TITAN EDR agent endpoints (Process, Network, AMSI, etc.) compiled in C++.

DOWNLOAD 'MASTER' REPOSITORY
>_ Execution Instructions:

1. Download the repository as a full .zip file and extract it.

2. Navigate to the extracted folder.

3. Right-click dashboard.exe and select "Run as Administrator". (Admin privileges are strictly required to hook into kernel-level ETW tracing).

03 / Layer 2 & 3: AI Brain & Threat Intel (Python)

This folder contains the Python Machine Learning engine, the JSON output parsers, and the backend API architecture.

DOWNLOAD 'CORRELATOR' REPOSITORY

Save all the .bat execution files inside the root correalator/ directory (the same level as the backend/ and frontend/ folders). Double-click the following scripts to launch the system components:

run_full_demo.bat

Executes the core Threat Analyzer pipeline and automatically launches the frontend web interface after a 3-second initialization delay.

run_sentinel_ai.bat

Wakes up the sentinel_ai_brain.py script. This loads the XGBoost .pkl model into memory to evaluate correlated telemetry events.

run_c_file.bat

Executes the custom c.py script for localized event processing and JSON output formatting.

Note: Ensure Python 3.12+ is installed with necessary libraries (pandas, scikit-learn, xgboost, flask) before executing the batch files.
